📝 Comandos útiles de Git

📌 Inicialización y configuración

  • git init - Inicializa un repositorio nuevo en la carpeta actual.
  • git clone <url> - Clona un repositorio remoto a tu máquina.
  • git config --global user.name "Tu Nombre" - Configura tu nombre en Git.
  • git config --global user.email "tu@email.com" - Configura tu email en Git.

📂 Estado y seguimiento de cambios

  • git status - Muestra el estado actual del repositorio.
  • git add <archivo> - Agrega un archivo específico al área de staging.
  • git add . - Agrega todos los archivos modificados al área de staging.
  • git restore <archivo> - Descarta cambios en un archivo (antes de hacer commit).

💾 Guardar cambios

  • git commit -m "mensaje" - Guarda los cambios con un mensaje.
  • git commit -am "mensaje" - Agrega y commitea archivos modificados (no nuevos).

🌿 Ramas (branches)

  • git branch - Lista las ramas.
  • git branch <nombre> - Crea una nueva rama.
  • git checkout <rama> - Cambia de rama.
  • git checkout -b <rama> - Crea y cambia a una nueva rama.

🔀 Fusión de ramas

  • git merge <rama> - Fusiona una rama con la actual.

🌐 Trabajo con repositorios remotos

  • git remote add origin <url> - Conecta tu repositorio local con uno remoto.
  • git push origin <rama> - Sube tus cambios al repositorio remoto.
  • git pull - Trae cambios del repositorio remoto y los fusiona.
  • git fetch - Trae cambios sin fusionarlos.

🕓 Historial

  • git log - Muestra el historial de commits.
  • git log --oneline - Historial resumido en una línea por commit.

⚠️ Deshacer cambios

  • git reset --hard - Elimina TODOS los cambios (usar con cuidado).
  • git revert <commit> - Deshace un commit creando uno nuevo.

🚀 Tips útiles

  • Usá mensajes de commit claros y cortos.
  • Hacé commits pequeños y frecuentes.
  • Usá ramas para nuevas funcionalidades.
  • Siempre hacé git pull antes de trabajar.

📚 Flujo básico de trabajo

  1. git add .
  2. git commit -m "mensaje"
  3. git push
